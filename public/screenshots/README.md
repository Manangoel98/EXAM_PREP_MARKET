# Screenshot Capture Guide

## Required Screenshots (Place in this folder)

Please capture the following screenshots from the **student app** (`app.nomoexam.com` or `localhost:3001`):

### 1. `dashboard-overview.png`
**Where:** `/dashboard` (main dashboard page)
**What to capture:** 
- Full dashboard view showing progress cards
- Study streak calendar
- Recent activities section
- Performance metrics

### 2. `ai-tutor-chat.png`
**Where:** `/nomoai` (AI tutor page)
**What to capture:**
- Active chat conversation with Nomo AI
- Show a detailed explanation with step-by-step breakdown
- Multiple message bubbles visible
- Chat input at bottom

### 3. `practice-test-list.png`
**Where:** `/practice-papers` (practice papers listing)
**What to capture:**
- Multiple practice tests visible
- Filters shown (difficulty, status)
- Performance badges on test cards
- Question counts and time estimates

### 4. `practice-test-active.png`
**Where:** During an active practice test
**What to capture:**
- Question with answer options
- Timer visible at top
- Question navigator sidebar
- Math input or multiple choice clearly visible

### 5. `test-results-summary.png`
**Where:** After completing a test (results page)
**What to capture:**
- Overall score prominently displayed
- Section-by-section breakdown
- Performance graphs/charts
- Time analysis

### 6. `flashcards-deck-list.png`
**Where:** `/flashcards` (flashcard decks page)
**What to capture:**
- Multiple flashcard decks displayed
- Subject organization
- Progress indicators on each deck
- Mastery vs needs practice stats

### 7. `flashcards-practice.png`
**Where:** During flashcard practice
**What to capture:**
- Active flashcard (front or back showing)
- Swipe/flip interface
- Session progress indicator
- Control buttons

### 8. `study-plan-overview.png`
**Where:** `/study-plan` (study plan page)
**What to capture:**
- Timeline/calendar view
- Daily tasks listed
- Progress tracking
- Target exam date visible

### 9. `analytics-performance.png`
**Where:** `/dashboard` or analytics section
**What to capture:**
- Score progression charts
- Subject-wise accuracy graphs
- Trend lines showing improvement
- Time management analytics

### 10. `exam-selection.png`
**Where:** `/exams` (exam selection page)
**What to capture:**
- Available exams (SAT, ACT, GRE, etc.)
- Exam cards with descriptions
- Selection interface
- Current exam highlighted

### 11. `profile-settings.png`
**Where:** `/profile` (profile/settings page)
**What to capture:**
- User profile information
- Notification settings
- Subscription status
- Customization options

---

## Screenshot Guidelines

- **Resolution:** Capture at high resolution (at least 1920x1080 desktop)
- **File Format:** PNG (preferred) or JPG
- **Naming:** Use EXACT names listed above (lowercase, with hyphens)
- **Content:** Make sure all text is readable and UI is clear
- **State:** Ensure the app is in a good "demo" state (populated with data, not empty)

## After Capturing

1. Place all screenshots in this folder (`/public/screenshots/`)
2. Make sure filenames match exactly (case-sensitive)
3. The ProductShowcase component will automatically display them
